# Auto Doc

Auto Doc helps teams turn under-documented GitHub and GitLab repositories into
reviewable, publishable documentation. It analyses source files, identifies weak
or missing documentation, generates docstring suggestions, scaffolds Sphinx
documentation, and supports publishing reviewed HTML output.

Use the hosted service:

https://autodoc.humanmetabolism.org

## What It Does

- Analyses GitHub and GitLab repositories
- Scans Python, JavaScript, TypeScript, and MATLAB source files (`.m`/`.matlab`)
- Generates reviewable docstring suggestions with OpenAI, Codex CLI, or Claude CLI
- Creates Python-only docstring insertion pull requests for GitHub repositories
- Creates Sphinx documentation with AutoAPI support
- Supports review-first publishing to GitHub Pages
- Provides an admin dashboard for repositories, runs, logs, and generated artifacts

## Local Development

Requirements:

- Python 3.11+
- `uv`
- Git
- An OpenAI API key or authenticated local AI CLI, if testing generation

Install dependencies:

```sh
uv venv
source .venv/bin/activate
uv sync --group dev --no-install-project
```

Create a `.env` file:

```env
ADMIN_PASSWORD=choose-a-strong-password
ADMIN_SECRET_KEY=choose-a-long-random-secret
OPENAI_API_KEY=your-openai-api-key
```

Run the app:

```sh
uv run uvicorn main:app --app-dir src --reload
```

The local app runs at `http://localhost:8000`.

- Admin dashboard: `http://localhost:8000/admin`
- FastAPI docs: `http://localhost:8000/docs`

## Docker

```sh
docker compose up --build
```

## Build the Docs Locally

```sh
env UV_CACHE_DIR=/tmp/uv-cache uv run --group docs sphinx-build -b html docs docs/_build/html-preview
```

Then open:

```text
docs/_build/html-preview/index.html
```
